# REAPER MCP 功能待办（调研版）

这份清单聚焦“音乐工程里高频且实用，但当前 MCP 还缺”的能力。  
每项都说明了：**功能是什么**、**为什么必须做**。

## P0（最高优先级）

### 1）路由 / Send / Receive / 硬件输出
- 是什么：
  - 创建/删除 send。
  - 枚举轨道 send/receive。
  - 设置 send 的音量、声像、静音及关键路由参数。
- 为什么必要：
  - 真实混音离不开总线、并行处理、sidechain。
  - 没有路由 API，就很难搭建可用的专业工程结构。

### 2）FX 链管理（Track FX + Take FX）
- 是什么：
  - 按名称添加/删除插件。
  - 列出 FX 链。
  - 启用/旁路插件。
  - 读取/设置插件参数（normalized）。
- 为什么必要：
  - 插件控制是 DAW 核心工作流。
  - 做人声链、母线链、stems 后处理、A/B 对比都依赖它。

### 3）自动化包络编辑
- 是什么：
  - 按名称获取包络（音量/声像/FX 参数）。
  - 插入、修改、删除包络点。
  - 批量写点并排序。
- 为什么必要：
  - 混音质量很大程度来自自动化（推子骑行、转场、效果动态）。
  - 只有静态参数，不足以完成可用的音乐工程编辑。

## P1（高价值）

### 4）Region 工作流 + Region Render Matrix
- 是什么：
  - 创建/查询/修改/删除 region。
  - 维护 region-track 渲染矩阵，用于批量导出。
- 为什么必要：
  - 交付场景高频：分段导出、分轨导出、批量交付。
  - 能把导出流程标准化、自动化、可复现。

### 5）录音准备控制（Rec Arm / Monitor / Input / Record Mode）
- 是什么：
  - 设置/读取轨道录音待命。
  - 设置/读取监听模式。
  - 设置/读取录音输入与录音模式。
- 为什么必要：
  - 一旦进入真实录音场景，这是刚需。
  - 能降低人工配置错误，支撑 AI 辅助录音流程。

### 6）Item 精细编辑参数
- 是什么：
  - Item 音量、淡入淡出、自动淡化。
  - Take 播放速率与音高。
- 为什么必要：
  - 这是编排与后期微调的日常操作。
  - 对清理衔接、时间修正、创意处理都非常关键。

## P2（进阶但非常实用）

### 7）Fixed Lanes / Comping 辅助能力
- 是什么：
  - 读取/设置 fixed lanes 相关轨道与 item 属性。
  - 控制 lane 播放状态（基础 comp 流程）。
- 为什么必要：
  - REAPER 现代编辑里，很多人声/乐器多录都用 lanes。
  - 能显著提高 comp 效率与可控性。

### 8）Razor Edit 辅助能力
- 是什么：
  - 读取/写入 razor edit 区域定义。
  - 基于 razor 范围执行批量精确编辑。
- 为什么必要：
  - 这是 REAPER 高效精修的关键工作流之一。
  - 可实现“局部、低副作用”的 AI 编辑。

## 建议实施顺序
1. 路由 / Send
2. FX 链管理
3. 自动化包络
4. Region + 渲染矩阵
5. 录音准备控制
6. Item 精细编辑参数
7. Fixed Lanes
8. Razor Edit 辅助
